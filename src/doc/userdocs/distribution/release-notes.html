This is a maintenance release that fixes several bugs present in Fedora 2.2.1. In addition, the release includes key improvements to the journaling and policy enforcement modules. See below for more detail.

  1. Bug Fixes

    • Bug #1701766: Clean up prompts/messages for "other" servlet container
    • Bug #1714089: HTTP Error Codes are not being returned correctly
    • Bug #1710801: HTTP Requests include bad HTTP basic Auth
    • Bug #1716106: demo soapclient fails to compile/run
    • Bug #1716636: addDatastream allows invalid values for checksumType
    • Bug #1720485: Null pointer running v2.2 installer
    • Bug #1740417: dcDates table not updated when DC datastream modified
    • Bug #1747186: FedoraAccessServlet hardcodes tiny buffer for datastreams
    • Bug #1771136: Whitespace added when modifying Inline XML
    • Bug #1773195: Incorrect XACML demo policies involving state
    • Bug #1785608: SQL rebuild sometimes fails to populate pidGen correctly
    • Bug #1791683: Installer creates one-jar doc directory
    • Bug #1816735: Flush and close missing in FieldSearchServlet
    • Bug #1832564: XSD dateTimes can have incorrect fractional seconds
    • Bug #1835449: Object-specific POLICY datastream ignored when using Oracle
    • Bug #1838750: improper handling of default namespace in batchModify
    • Bug #1844803: Incorrect protocol/port in DefaultExternalContentManager
    • Bug #1848092: Incorrect handling of dates before 1CE
    • Bug #1848095: RelsExtValidator rejects valid UTC dateTimes
    • Bug #1853905: Installer fails if FEDORA_HOME contains a space
    • Bug #1863382: Lockup after frequent dissemination requests
    • Bug #1878847: Problem with utf-8 characters in datastream
    • Bug #1927861: ExecUtility calls can fail if command has a space

  2. Additional Improvements

    • Policy Enforcement
      • Feature #1743769: Multiple Owner id Support
        Fedora's bundled XACML engine can now act on policies involving objects that have more than one owner. See Fedora Authorization with XACML for more information on configuring and using this feature.

    • Journaling
      The following improvements enable more sophisticated high availability configurations with Fedora. See the Journaling Guide for more information on configuring and using these features.
      • RMI Support - Journal leaders/followers can now be configured to communicate via RMI.
      • Multicast Support - Journaling now supports multiple followers via disk, RMI, or combination.
      • Read-only Mode - Formerly, when configured as a follower, a Fedora repository would give an error for any API-M call. Now, a follower will allow calls to API-M methods that do not make changes to the repository.

    • Miscellaneous
      • Fedora's out-of-box SSL server certificate (not intended for production use) has been updated with a later expiration date, for convenience.
